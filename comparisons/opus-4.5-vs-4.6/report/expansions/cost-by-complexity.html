At complex tasks, Opus 4.6 uses dramatically fewer fresh input tokens per task (27 vs 343) despite producing 34% more output (4,398 vs 3,277). This reflects its strategy of batching more work into each API call (lower requests per task: 24.8 vs 30.2 for complex tasks; 6.8 vs 9.5 overall), which means the prompt is re-sent fewer times. Combined with more effective cache utilization, the output cost premium is more than offset by input savings.

Caveat: These averages come from organic sessions with different task mixes per model. Some of the per-complexity gap may reflect session-level factors (e.g., caching benefits accumulate within longer sessions).
