At complex tasks, Opus 4.6 uses dramatically fewer fresh input tokens per task (29 vs 1,352) while producing 22% less output (3,261 vs 4,170). Despite using similar request counts per task at the complex level (25.3 vs 26.2), Opus 4.6 achieves much more effective cache utilization. Combined with more effective cache utilization, the output cost premium is more than offset by input savings.

Caveat: These averages come from organic sessions with different task mixes per model. Some of the per-complexity gap may reflect session-level factors (e.g., caching benefits accumulate within longer sessions).
