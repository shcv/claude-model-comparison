At complex tasks, Opus 4.6 uses dramatically fewer fresh input tokens per task (33 vs 1,479) while producing 9% more output (4,350 vs 3,990). Despite using similar request counts per task at the complex level (28.2 vs 26.5), Opus 4.6 achieves much more effective cache utilization. Combined with more effective cache utilization, the output cost premium is more than offset by input savings.

Caveat: These averages come from organic sessions with different task mixes per model. Some of the per-complexity gap may reflect session-level factors (e.g., caching benefits accumulate within longer sessions).
