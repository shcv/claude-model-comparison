Executive Summary

This report presents a comparative behavioral analysis of two Claude Code models—Opus (550 tasks) and Fudge (295 tasks)—drawn from real coding sessions during the Claude Code Early Access Program. The analysis combines automated quantitative extraction, LLM-powered qualitative assessment, and tool-call pattern analysis to surface meaningful behavioral differences.

3. Parallelization — Directive Compliance

Neither model ever issues multiple tool calls in a single assistant message (0 out of ~9,000 messages). "Parallelism" manifests exclusively through sequential subagent dispatch. When the user explicitly requests parallel execution, the models diverge sharply in interpretation.

6. Behavioral Model Summary

Opus Profile

Conservative subagent use (9.8%). Relies on explicit user requests for delegation (44%). Prefers heavyweight general-purpose agents (50% of calls). Rarely plans autonomously (1.1%). Stronger on trivial tasks. Interprets "parallel" as "multiple sequential agents."

Fudge Profile

Aggressive subagent use (29.1%), almost entirely autonomous (90%). Strongly prefers lightweight Explore agents (72%). Frequently plans (11.5%), autonomously 91% of the time. Stronger on complex tasks. Correctly implements background execution for parallel requests.

Why Fudge's strategy works for complex tasks: Heavy upfront exploration builds deeper codebase understanding before implementation. Keeping implementation in-house avoids "telephone game" delegation errors. Frequent autonomous planning catches issues before code is written. The tradeoff is measurable overhead on trivial tasks where this investment doesn't pay off.

7. Methodology

Analysis Pipeline

Raw Sessions.jsonl files
→
Extract Tasksdeep data
→
Classifycomplexity
→
LLM Analysisqualitative
→
Reportsquantitative + behavioral

Each task was assessed using Claude Haiku with a structured prompt evaluating alignment, sentiment, completion, autonomy, scope, iteration, and error recovery. "Ground truth" for user satisfaction was inferred from the user's subsequent message. Complexity classification used automated signals (tool count, file count, duration). Behavioral analysis used pattern matching on raw tool call data.

Limitations

Single-user sample (n=1). Task boundaries determined by user messages, not semantic completion. "Autonomous" detection relies on keyword matching. LLM judgments are subjective proxy measures. No statistical significance testing. The "next user message" is an imperfect proxy for actual satisfaction.
